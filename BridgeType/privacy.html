BridgeType Privacy Policy

Last updated: June 7, 2026

1. Who this policy applies to

This Privacy Policy applies to the BridgeType Android app and the BridgeType website published through GitHub Pages.

  • Android application ID: com.fj.BridgeType
  • Website: https://farzadj.github.io/BridgeType/

BridgeType is a translator keyboard app. It does not require a BridgeType account and does not depend on a custom BridgeType server for its core translation features.

2. Privacy-first keyboard design

BridgeType is designed to translate messages without unsafe access to private messenger screens.

  • No AccessibilityService is used.
  • No chat screen scraping is used.
  • No overlay is used to inspect another app's private messages.
  • No clipboard monitoring is used.
  • BridgeType does not collect or store your chat messages.

The app uses normal Android keyboard and text-sharing APIs. This means the user controls what text is typed, pasted, dictated, selected, shared, translated, copied, or inserted.

3. Text BridgeType can process

BridgeType can process text only when you explicitly provide it through app features such as:

  • text you type in the BridgeType keyboard draft box
  • text you type directly into the currently focused messenger text field with the keyboard
  • text you paste into Receive mode using the paste button
  • text you select and send to BridgeType through Android ACTION_PROCESS_TEXT
  • text you share to BridgeType through Android ACTION_SEND
  • speech you start by tapping the mic button

BridgeType does not scan full conversations and does not automatically read messages from WhatsApp, Telegram, Signal, Instagram, SMS, or other apps.

4. Translation and language detection

BridgeType uses Google ML Kit on-device Translation and Language Identification for the current MVP.

  • Language models may need to be downloaded before a language pair can be translated.
  • Internet access may be used to download ML Kit language models.
  • After the required model is available, translation runs on the device.
  • Translation text is used in memory to produce the requested translation and is not saved by BridgeType as chat history.

Model availability, selected languages, voice settings, keyboard style, keyboard color, saved language pairs, and feature usage counters may be stored locally so the app can remember your settings.

5. Voice recognition

BridgeType can use Android speech recognition after you tap the mic button. This requires microphone permission.

  • BridgeType starts voice recognition only after your explicit action.
  • BridgeType does not continuously listen in the background.
  • BridgeType does not store voice recordings.
  • Speech recognition behavior can depend on your device, Android version, installed speech services, and Google settings.

Depending on the speech recognizer available on your device, audio may be processed by a device speech service such as Google speech recognition. Those services operate under their own privacy policies and device settings.

6. Clipboard use

BridgeType does not monitor the clipboard and does not watch for clipboard changes.

Clipboard access can happen only after an explicit user action, for example pressing a paste button, pressing a copy button, or using normal Android text editing controls.

7. Local app data

BridgeType stores app settings locally on your device. This can include:

  • default source and target language choices
  • saved language pairs
  • keyboard language layout selection
  • theme, color, and keyboard style choices
  • voice timing settings
  • downloaded model status indicators
  • daily Live mode usage counters for free-plan limits
  • Pro entitlement status when applicable

Draft message text and translated chat text are not stored as a message history by BridgeType. They may exist temporarily in app memory while you are using the keyboard or translation screen.

8. Analytics

BridgeType may use Firebase Analytics to understand feature usage, reliability, and conversion behavior. Analytics events are designed to avoid private chat content.

Analytics can include event metadata such as:

  • app open and screen/tab usage
  • keyboard mode changes such as Compose, Receive, or Live
  • selected source and target language codes
  • translation request, success, and failure outcomes
  • coarse text length buckets, not the text itself
  • language model download status
  • voice recognizer start, success, timeout, or error outcomes
  • keyboard theme, color, spacing, and settings changes
  • free or Pro entitlement status

BridgeType analytics does not send message text, translated text, selected incoming text, voice transcripts, clipboard contents, chat-screen content, contact details, phone numbers, or messenger app names.

9. Subscriptions and payments

BridgeType may offer optional Pro access through Google Play subscriptions in production releases.

  • Payments are processed by Google Play.
  • BridgeType does not receive your full payment card details.
  • BridgeType may receive purchase and entitlement status needed to unlock, restore, or limit Pro features.
  • Subscription cancellation, renewal, refund, and tax handling are managed through Google Play.

10. Network usage

BridgeType can use network access for:

  • downloading ML Kit language models
  • Firebase Analytics event reporting, when Firebase is configured
  • Google Play subscription and app distribution services, when Pro access is available
  • GitHub Pages website hosting

The current app does not require a custom BridgeType backend server for translation.

11. Third-party services

BridgeType may rely on the following third-party services:

  • Google ML Kit Translation and Language Identification
  • Android SpeechRecognizer and installed speech recognition services
  • Firebase Analytics, when configured
  • Google Play and Google Play Billing, when subscriptions are available
  • GitHub Pages for this website

These services operate under their own privacy policies and platform settings.

12. Website hosting

The BridgeType website is hosted on GitHub Pages. When you visit the website, GitHub may process standard hosting data such as IP address, browser information, request logs, and security-related information according to GitHub's policies.

13. Your choices

  • You can enable or disable the BridgeType keyboard in Android keyboard settings.
  • You can choose when to paste, copy, dictate, translate, share, or insert text.
  • You can remove downloaded language models through app settings where available or through Android app storage controls.
  • You can deny microphone permission if you do not want to use voice recognition.
  • You can clear app data or uninstall BridgeType to remove locally stored settings, subject to Android backup behavior.
  • You can manage subscriptions through Google Play if Pro access is available.

14. Children's privacy

BridgeType is a general communication utility. It is not designed to knowingly collect personal information from children through a BridgeType account system. Parents or guardians should review device keyboard, microphone, subscription, and privacy settings before allowing a child to use the app.

15. Changes to this policy

This Privacy Policy may be updated when BridgeType features, data flows, third-party services, or legal requirements change. The updated version will be published on this page.

16. Contact

For project questions, bug reports, support, or privacy-related requests, use the project repository or support page:

  • GitHub repository
  • GitHub issues
  • Support page
